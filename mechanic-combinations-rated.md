# Mechanic Combination Ratings — Meier Lens

*Sid Meier's test applied to each combination: "Does pairing these two mechanics create a new interesting decision that neither mechanic produces alone?"*
*Ratings: A = Strong (new interesting decision named), B = Possible, C = Weak*

---

## A — Strong (35)

**2. Rotation + Copying / Cloning**
Decide whether to rotate now (optimizing current fit) or clone first (preserving the original orientation for later), creating a timing tension between immediate efficiency and future flexibility.

**4. Pile Matching + Copying / Cloning**
Decide whether to spend the clone resource to hedge multiple pile targets or commit the original, making every match a risk-calibration choice.

**8. Pattern Completion + Trick Taking**
The same card often advances your private pattern or wins the trick but not both, forcing a constant choice between personal progress and competitive dominance.

**12. Rotation + Card Capture**
The orientation you choose before capturing determines which values apply, so every capture is simultaneously a configuration decision.

**22. Forced Move Detection + Race to Threshold**
Reading that your opponent is in a forced move forces a choice: trigger it now for tempo, or delay and accumulate more threshold progress first.

**28. Teleportation + Rule Rewriting**
Decide whether to teleport under current rules (safe, immediate) or rewrite a rule first (better destination) at the cost of the action that could have been the teleport itself.

**32. Binary Elimination + Investment & Return**
Every elimination also destroys its future return potential, so the player must weigh a certain small gain now against an uncertain larger gain from keeping the option alive.

**48. Card Capture + Action Points**
Action point costs make capturing expensive relative to other actions, forcing the player to judge which cards are worth the AP and which should be left for opponents.

**51. Binary Elimination + Role Swap**
Eliminating an option may destroy the ability or role you'll need after a swap, so the player must predict their future role before deciding what to cut.

**54. Action Points + Constraint Inversion**
Decide whether to spend AP under current constraints (predictable cost) or spend AP to invert a constraint (reshaping all future costs), making every expenditure a choice between present and meta-game.

**59. Scale Change + Hidden Information Deduction**
Zooming out reveals spatial context but obscures detail needed for deduction; zooming in enables precise reasoning but hides the bigger picture, so scale transitions must be timed around information needs.

**65. Chase & Flee + Trick Taking**
Winning a trick may force you to lead next — exposing your position in a chase context — so you must decide whether the trick win is worth the positional vulnerability.

**68. Constraint Propagation + Consumption**
Consuming a resource relaxes or triggers constraint propagation elsewhere, so consumption order becomes a question of which constraint wave you can and cannot survive.

**76. Cooldown + Catch-Up Mechanic**
A trailing player must choose between firing a powerful ability now (triggering cooldown) or waiting for the catch-up bonus to compound, since both paths reach parity but at different speeds.

**96. Constraint Satisfaction + Time Rewind**
Rewinding to undo a constraint violation is always available but costs turns, so the player must judge whether a risky forward path is worth more than the rewind cost to explore a safer branch.

**97. Fog of War + Scarcity Management**
Spending scarce resources to reveal fog directly competes with spending them on actions, making visibility itself a rationed commodity with real opportunity cost.

**100. Deduction Grid + Lever Toggle**
Toggling a lever changes puzzle state and may invalidate completed deductions, forcing the player to judge whether current logical certainty is worth preserving or whether toggling opens a better solution path.

**104. Zone Control + Key and Lock**
A key only has value if you can hold the zone it unlocks; the player must decide whether to commit control resources to the current zone or gamble them on the unlock, knowing they can't do both.

**107. Deduction + Forced Move Detection**
Deducing that an opponent has a forced move turns deduction into a tactical weapon: reveal it now (triggering the forced move) or conceal it (gathering more information).

**109. Deck Cycling + Hidden Information Deduction**
Cycling the deck builds probabilistic knowledge of upcoming cards, so the decision is how much to act on incomplete current information versus wait for the cycle to confirm it — certainty against tempo.

**113. Pathfinding / Navigation + Layered Planes**
Moving to a different plane changes which paths are available across all planes simultaneously, so every navigation decision also restructures the entire movement graph.

**117. Deduction + Pressure Plate**
Standing on a pressure plate before fully deducing what it activates is a speed-versus-certainty choice with potentially irreversible consequences.

**131. Layered Planes + Key and Lock**
A key obtained on one plane may only function on a lock on another plane, forcing coordinated traversal across planes rather than local optimization.

**134. Time Rewind + Skill Tree / Ability Unlocking**
Rewinding erases recently unlocked abilities alongside mistakes, so the player must decide how far back to rewind — preserving hard-won unlocks at the cost of a dirty state, or rewinding fully for a clean but ability-poorer board.

**139. Deduction Grid + Choice Branching**
Each branch generates new evidence for the deduction grid, so the player must decide whether to branch speculatively (generating information) or deduce further first (reducing wasted moves).

**141. Scale Change + Sequence Locking**
A sequence locked at one scale may be the wrong sequence at another, so the player must decide when in the sequence to change scale — too early wastes locked steps, too late closes the window.

**153. Hidden Information Deduction + Resource Accumulation**
Spending resources to reveal hidden information creates a direct market: how much is worth paying to confirm what can be inferred for free, making information a priced commodity.

**162. Gravity Shifting + Action Points**
A gravity shift repositions all objects at once; the player must decide whether a shift that incidentally moves multiple pieces is worth more than spending the same AP on targeted individual actions.

**165. Zone Control + Line of Sight**
Zone selection is simultaneously a visibility decision — controlling a zone opens certain sight lines and closes others, forcing the player to weigh offensive views against defensive blind spots.

**171. Constraint Propagation + Pressing Luck**
Pressing luck extends a chain that also propagates constraints; a lucky result may force a constraint cascade the player didn't want, so the decision to press is about acceptable constraint states, not just probability.

**184. Key and Lock + Constraint Inversion**
Inverting a constraint may flip what counts as a key or lock, so the player must decide whether to use a key under current rules or invert first — risking making the key useless while potentially unlocking more.

**188. Hidden Information + Binary Elimination**
Eliminating a hidden option permanently destroys information that might have been revealed later, so the player must weigh a certain small gain against the unknown value of what they're cutting forever.

**195. Hidden Information + Chain Scoring**
Playing into a chain commits to a score trajectory you can't fully evaluate; the decision is whether to extend an unknown chain or break it for a certain smaller score.

**196. Memory Match + Constraint Inversion**
Inverting a constraint mid-game (e.g., "match opposites instead of pairs") can invalidate an opponent's memory advantage, creating a strategic timing decision about when to invert.

**197. Consumption + Constraint Inversion**
Consuming a resource to invert a constraint reshapes all future consumption decisions, so the player must evaluate not just the immediate value but how inversion changes the entire remaining resource economy.

## C — Weak (10)

10. Grid Movement + Trick Taking — mechanics coexist but don't interact to create new decisions.
14. Pull Block + Placement — Pull Block is a placement sub-variant; no new decision layer.
16. Stacking + Tap to Cycle — Tap to Cycle is a UI gesture; adds no new decision to stacking.
23. Chase & Flee + Stacking — mechanics run in parallel without meaningful interaction.
130. Maze Traversal + Undo — Undo in a maze just removes the challenge of dead ends rather than creating decisions.
185. Line of Sight + Letter Selection — LoS filters which letters are selectable; this is UI gating, not a new decision.
198. Teleportation + Anagram Solving — teleporting to collect letters is just navigation serving word-building; no new tension between the mechanics.
10. Grid Movement + Trick Taking — see above.
83. Pull Block + Bluffing — bluffing removed in filter; listed in error.
160. Signal Detection + Trading — trading removed in filter; listed in error.
